KJ CLIController API Guide

Integration guide for the Local AI CLI Gateway API

Overview

CLIController provides a gateway API that is fully compatible with the OpenAI format. When you call this API, the system automatically spawns a subprocess that drives the local CLIs running on the host machine and returns results quickly.

Supported Models / CLIs:

You can set the "model" field in the request body to one of the following values:

claude gemini openai

* Note: "openai" activates the local Codex CLI on your device.

Available Endpoints

All routes except this documentation page require a token authentication header.

Method Endpoint URL Description
POST /v1/chat/completions Send prompts and receive text responses from the corresponding CLI
GET /api/providers List the local CLIs and their availability status
POST /api/upload Upload attachment files (images, text) to the temporary directory
GET /api/guide View this API integration guide page

Sending a Chat Completion

Main endpoint: POST /v1/chat/completions. Two required headers: 

Authorization: Bearer my-secret-lan-token
Content-Type: application/json

The JSON payload structure includes:

  • model: The model to call ("claude" | "gemini" | "openai")
  • messages: An array of conversation messages, each with role and content
  • timeout: CLI command execution timeout (default: 90 seconds)
  • cwd: (Optional) The absolute working directory path on the host where the CLI subprocess should execute. If not provided, it will automatically default to the parent directory of the first file attachment.

cURL Example

You can run the following cURL command from your terminal to test a response from the Claude CLI:

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer my-secret-lan-token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude",
    "messages": [
      {
        "role": "user",
        "content": "Say hello in Rust"
      }
    ]
  }'

PowerShell Example

You can run this query using PowerShell's native Invoke-RestMethod cmdlet:

$headers = @{
    "Authorization" = "Bearer my-secret-lan-token"
    "Content-Type" = "application/json"
}

$body = @{
    model = "claude"
    messages = @(
        @{ role = "user"; content = "Say hello in Rust" }
    )
    timeout = 120
    cwd = "F:\AntiGravity\testgame" # (Optional) Workspace path
} | ConvertTo-Json -Depth 8 -Compress

$response = Invoke-RestMethod -Uri "http://localhost:8080/v1/chat/completions" -Method Post -Headers $headers -Body $body
$response.choices[0].message.content

Python Integration Example

Use Python requests to quickly embed this capability into the AI services of other projects:

import base64
import requests

def ask_local_cli(model, prompt, system_prompt=None, image_path=None, cwd=None):
    url = "http://localhost:8080/v1/chat/completions"
    headers = {
        "Authorization": "Bearer my-secret-lan-token",
        "Content-Type": "application/json"
    }
    
    # Assemble the prompt and handle base64 image if present
    user_content = [{"type": "text", "text": prompt}]
    
    if image_path:
        with open(image_path, "rb") as img_file:
            b64_str = base64.b64encode(img_file.read()).decode("utf-8")
        ext = image_path.split(".")[-1].lower()
        mime = f"image/{ext}" if ext in ["png", "jpg", "jpeg", "webp", "gif"] else "image/jpeg"
        
        user_content.append({
            "type": "image_url",
            "image_url": {"url": f"data:{mime};base64,{b64_str}"}
        })
        
    messages = []
    if system_prompt:
        messages.append({"role": "system", "content": system_prompt})
    messages.append({"role": "user", "content": user_content})
    
    payload = {
        "model": model,
        "messages": messages,
        "timeout": 120
    }
    if cwd:
        payload["cwd"] = cwd
    
    response = requests.post(url, json=payload, headers=headers)
    response.raise_for_status()
    return response.json()["choices"][0]["message"]["content"]

# Usage:
# text = ask_local_cli("claude", "Write a bubble sort algorithm")
# print(text)

Sending File Attachments

To attach large non-image text documents (such as .txt, .py, .csv), the simplest approach is to read the file contents in your own script and insert that text directly into the prompt you send: 

Please analyze the structure of this config file:

[Attached file: config.json]
{
  "port": 8080,
  "token": "my-secret-lan-token"
}

Multi-Agent Orchestration

The real power of this gateway is using one desktop AI agent as a conductor that chats with you, makes a plan, and then delegates step-by-step subtasks to the other local CLIs — so the conductor and the workers keep a shared understanding across each step. This works no matter which agent you drive (Gemini/Antigravity, Claude Code, or Codex), and it stays portable across machines.

How shared context works

The gateway is stateless and each CLI runs as a fresh one-shot process, so the shared memory lives on disk: a ledger file AGENT_LOG.md in your working directory. Every delegated worker reads it first and appends what it did; the conductor reads it before each next step. Pass your project folder as the cwd field so workers run inside it with real file access.

Suggested roles: gemini main conductor  ·  claude senior technical advisor (use sparingly)  ·  openai broad context / overview.

Setup (one time)

  1. Install & authenticate the CLIs you want: claude, agy (Antigravity), codex.
  2. Start the gateway, then confirm GET /api/providers shows them as available: true.
  3. Copy the protocol files from the repo's orchestration/ folder into the root of the project you want to work in: ORCHESTRATION.md plus the pointer for your conductor — CLAUDE.md (Claude Code), AGENTS.md (Codex), or GEMINI.md (Antigravity).
  4. (Optional) Put consult.py on your PATH for a quick delegation command on any machine.
  5. Start chatting with your agent. Paste the bootstrap prompt below as your first message (or rely on the pointer file the agent reads automatically).

Bootstrap prompt (paste as your first message)

This single message tells any agent it can use the multi-agent panel through CLIController:

You can orchestrate other local AI agents through the KJCLIController gateway
(OpenAI-compatible) at http://localhost:8080, token "my-secret-lan-token".
Models: claude = Claude Code, gemini = Gemini/Antigravity, openai = Codex.
Check availability at GET /api/providers.

To collaborate across steps, keep a shared ledger AGENT_LOG.md in the working
directory: read it before each step, and instruct every delegated worker to read
it first and append a dated summary (what it did, key decisions, files changed).
Pass the workspace as the "cwd" field so workers run inside it.

Delegate with:  python consult.py --model  --prompt "..."
(or --all to ask the whole panel), or any OpenAI SDK with
base_url = http://localhost:8080/v1.

Roles: gemini = main conductor, claude = senior technical advisor (use sparingly),
codex = broad context/overview. Use paths relative to the workspace; never hardcode
absolute paths. If a provider is unavailable, skip it and continue.

Full protocol: read ORCHESTRATION.md in the working directory.

* Portability: always use paths relative to the workspace, and never assume a CLI is installed — check /api/providers and degrade gracefully. See ORCHESTRATION.md for the full protocol.
