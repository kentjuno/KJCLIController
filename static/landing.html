One API. Every Local AI CLI.

KJ CLIController is a blazing-fast Rust gateway that wraps Claude Code, Gemini, and Codex CLIs behind a single OpenAI-compatible endpoint — fully local, fully secure, zero cloud round-trips.

Get Started View on GitHub

Everything You Need. Nothing You Don't.

⚡

OpenAI-Compatible Endpoint

Drop CLIController into any project already talking to OpenAI. POST /v1/chat/completions speaks the exact same protocol — swap the base URL and you're done.

🔀

Multi-Model Subprocess Router

Spawns claude, agy, and codex as native OS processes. No SDKs, no HTTP round-trips to the cloud — just your local binaries doing the work they were built for.

💬

Dynamic @-Mention Autocomplete

The built-in dark-mode chat dashboard lets you switch models mid-conversation with an @claude, @gemini, or @openai mention — with real-time autocomplete suggestions.

🔒

Constant-Time Bearer Auth

Every request is gated by a cryptographically safe, constant-time token comparison — eliminating timing side-channels and keeping your local network gateway locked tight.

🖼️

Image & File Attachment Pipeline

Attach images and documents directly in chat or via the API. Files are encoded as base64 data-URLs and routed seamlessly to whichever multi-modal CLI you've selected.

🏠

Pure Local-First Routing

Your prompts never leave the machine. CLIController routes exclusively over your local network — no external cloud APIs, no telemetry, no data leaving your control. Ever.

How It Works

A single Rust binary sits between your clients and your local AI CLIs. Incoming OpenAI-shaped requests are authenticated, parsed, and dispatched as subprocesses — responses are streamed back in real time over the same connection.

Supported Providers

claude Anthropic Claude Code CLI — powerful agentic coding with full filesystem context.
gemini Google Gemini / Antigravity CLI — next-generation multi-modal reasoning, locally routed.
openai OpenAI Codex CLI — battle-tested code generation running entirely on your hardware.

Up in Seconds

# Clone and launch the gateway
git clone https://github.com/clicontroller/clicontroller.git
cd clicontroller
cargo run --release
# Open the chat dashboard → http://localhost:8080/
# Call the OpenAI-compatible endpoint from any HTTP client
curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer my-secret-lan-token" \
  -d '{
    "model": "claude",
    "messages": [
      { "role": "user", "content": "Refactor this function to use iterators." }
    ]
  }'

API Reference

Method Endpoint Auth Description
POST /v1/chat/completions Bearer token OpenAI-compatible chat endpoint. Route to any provider via the model field. Supports streaming, images, and file attachments.
GET /api/providers Bearer token Returns the list of available CLI providers currently registered and reachable on this host.
POST /api/upload Bearer token Upload images or documents. Files are stored as base64 data-URLs and returned as attachment references for use in chat messages.
GET /api/guide None Interactive usage guide rendered in the built-in dark-mode dashboard. No auth required for local self-documentation.

Claude Code Skill

Install the bundled “Consult the CLIs” skill and just tell your Claude Code agent “tham vấn mấy ông CLI” (or “consult the other models”) — it routes your question to the local Claude, Gemini, and Codex CLIs and synthesizes their answers for you.

🗣️

One phrase, three minds

Name one model or ask the whole panel. The agent picks the best CLI for the task — or queries all three in parallel and reconciles consensus vs. disagreement.

🧩

Zero-dependency client

A single stdlib-only Python script does the dispatching — no pip installs, works the same on Windows, macOS, and Linux.

🛡️

Pitfalls handled

Auto-fixes Windows shell quote-mangling on inline curl and folds system prompts for the Codex CLI, which rejects the --system flag.

# Install into your Claude Code skills directory
cp -r skills/consult-local-clis ~/.claude/skills/

# …then in a Claude Code session, just say:
#   "tham vấn mấy ông CLI xem nên chọn Postgres hay SQLite"
#   "get a second opinion from Codex on this function"

# Or call the client directly:
python skills/consult-local-clis/scripts/consult.py --all --prompt "What is a mutex?"

Ships in the repo under skills/ — grab the folder or the packaged .skill bundle. See the README for full usage.
