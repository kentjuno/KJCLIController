# KJ CLIController: Local AI CLI Gateway

[![License: MIT](https://img.shields.io/github/license/kentjuno/KJCLIController?color=blue)](LICENSE)
[![Latest Release](https://img.shields.io/github/v/release/kentjuno/KJCLIController?sort=semver&color=brightgreen)](https://github.com/kentjuno/KJCLIController/releases)
[![Live Demo](https://img.shields.io/github/deployments/kentjuno/KJCLIController/github-pages?label=live%20demo)](https://kentjuno.github.io/KJCLIController/)
[![Made with Rust](https://img.shields.io/badge/Made%20with-Rust-orange?logo=rust&logoColor=white)](https://www.rust-lang.org/)

A secure, local network gateway written in Rust that exposes local AI Command Line Interfaces (CLIs) — Anthropic's **Claude Code**, Google's **Gemini/Agy CLI**, and OpenAI's **Codex CLI** — through a unified, OpenAI-compatible HTTP API endpoint. It ships with a modern, dark-mode SPA Chat Dashboard, a marketing landing page, an optional Windows system-tray mode, and a **multi-agent orchestration** toolkit.

> 🎼 **New — Multi-Agent Orchestration:** turn *any* desktop agent into a conductor that delegates step-by-step subtasks to the other local CLIs, keeping a shared `AGENT_LOG.md` ledger between them. Just hand an agent the `http://localhost:8080/agents` link. See **[ANNOUNCEMENT.md](ANNOUNCEMENT.md)** and the [Multi-Agent Orchestration](#multi-agent-orchestration) section below.

## Key Features

- **OpenAI-Compatible Chat Completions**: Exposes the standard `/v1/chat/completions` endpoint, allowing you to drop it directly into existing AI libraries or applications (e.g. the OpenAI SDK or custom Python agents).
- **Multi-Model Subprocess Router**: Automatically spawns and pipes subprocess prompts directly to `claude`, `agy`, and `codex` command binaries natively.
- **Dynamic Mentions Autocomplete**: The web chat UI supports triggering model selectors by typing `@` (e.g. `@claude`, `@gemini`, `@agy`, `@openai`, `@codex`), autocompleting, dynamically routing the prompt to the requested model, and highlighting message badges.
- **Local Security Tokens**: Secured by customizable Bearer Token authorization headers to prevent unauthorized local network users from exhausting your CLI quota.
- **Image/Attachment Pipeline**: Automatic parsing of base64 image data-URLs and text file attachments, mapping them to the specific parameters supported by each CLI binary.
- **Generated File Watcher**: Automatically monitors the `outputs/` folder during a CLI run, capturing any image/file outputs generated by the AI (e.g. by OpenAI Codex CLI), isolating them, and serving them via the Web UI.
- **Pure Local Routing**: Strictly executes local CLI commands. No external cloud HTTP REST APIs are called, keeping your code execution completely local first.
- **Multi-Agent Orchestration**: Let one conductor agent delegate subtasks to the other CLIs and share context across steps via an `AGENT_LOG.md` ledger in a shared workspace (passed as `cwd`). Orchestrator-agnostic and portable across machines.
- **Agent Bootstrap Endpoints**: `GET /agents` serves a machine-readable orchestration protocol (markdown), and `GET /consult.py` serves a zero-dependency delegation helper any agent can download with one `curl`.
- **Resilient Delegation**: The bundled `consult.py` client auto-retries transient failures (empty replies, HTTP 5xx, network blips) while skipping non-retryable quota/auth errors.
- **System Tray (Windows)**: Release builds run hidden in the system tray with a menu to open the UI, tail logs live, and toggle "Start with Windows".

## Repository Structure

- `src/main.rs`: Entry point. Launches the Axum web server, configures CORS, routes file uploads, and acts as the API endpoint controller.
- `src/config.rs`: Handles loading configuration from `config.json` or falling back to environment variables.
- `src/auth.rs`: Middleware validating Bearer tokens in constant time.
- `src/registry.rs`: Handles listing of providers, availability caching (60s TTL), and prompt dispatching.
- `src/tray.rs`: Windows-only system-tray integration (compiled in release builds): tray icon, menu, "Start with Windows" toggle.
- `src/cli/`:
  - `resolve.rs`: Resolves local system paths to executable CLI binaries, including Windows `.cmd` paths.
  - `validate.rs`: Guards prompt size (max 100KB), attachment limits, and path traversal vulnerabilities.
  - `claude.rs`: Adapter for Claude Code CLI subprocess execution.
  - `gemini.rs`: Adapter for Google Agy CLI subprocess execution.
  - `codex.rs`: Adapter for OpenAI Codex CLI subprocess execution.
- `static/`: Embedded frontend assets — `landing.html` (homepage), `index.html` (chat dashboard), `guide.html` (API guide), `agents.md` (agent-readable orchestration protocol), plus CSS/JS.
- `orchestration/`: Tool-agnostic orchestration protocol (`ORCHESTRATION.md`) and conductor pointer files (`CLAUDE.md`, `AGENTS.md`, `GEMINI.md`) to drop into your project.
- `skills/consult-local-clis/`: A Claude Code skill + the `consult.py` delegation client (also served at `/consult.py`).

## Quickstart

### 1. Prerequisites
Ensure that you have your target CLI tools installed and authenticated on your host system:
- **Claude Code**: authenticated via `claude` command.
- **Antigravity CLI (Agy)**: authenticated via `agy` command.
- **OpenAI Codex**: authenticated via `codex` command.

### 2. Compile & Run the Gateway
Clone this repository and compile the Rust project:
```cmd
cd f:\AntiGravity\CLIController
cargo run --release
```
On initial startup, it will generate a `config.json` file in the root directory.

### 3. Configuration (`config.json`)
You can modify ports and tokens inside `config.json`:
```json
{
  "token": "my-secret-lan-token",
  "port": 8080,
  "temp_dir": "./temp_uploads",
  "output_dir": "./outputs",
  "openai_api_key": null
}
```

### 4. Run in the System Tray (Windows)
Release builds run **hidden in the system tray** (no console window). Double-click `clicontroller.exe`, then use the tray icon to open the UI, tail logs live, or enable **Start with Windows**. Debug builds (`cargo run`) keep a console for development.

## Web UI Chat Dashboard
Open your browser and navigate to:
```
http://localhost:8080/chat
```
(`http://localhost:8080/` now serves the landing page.)

1. Input your configured authentication token (default: `my-secret-lan-token`).
2. Type your message in the chat input.
3. **Mention Trigger**: Type `@` to open the CLI model autocomplete popover. Choose a model using `Arrow keys` + `Enter`/`Tab` or by clicking. The prefix (e.g. `@claude `) will be stripped before submission, routing the call dynamically.

## API Reference

### 1. Chat Completions (`POST /v1/chat/completions`)

#### Headers
```http
Authorization: Bearer <your-token>
Content-Type: application/json
```

#### JSON Payload Schema
- `model`: Target CLI provider: `"claude"`, `"gemini"`, or `"openai"`.
- `messages`: Array of role/content message maps (standard OpenAI formatting).
- `timeout`: Maximum subprocess execution timeout (seconds). Defaults to `90`.
- `cwd` (optional): Absolute directory path on the host where the CLI subprocess should execute. If not provided, it will automatically default to the parent directory of the first file attachment.

#### Standard Request Example
```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer my-secret-lan-token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude",
    "messages": [
      {
        "role": "user",
        "content": "Write a quicksort function in Rust"
      }
    ],
    "timeout": 120
  }'
```

#### PowerShell Request Example
```powershell
$headers = @{
    "Authorization" = "Bearer my-secret-lan-token"
    "Content-Type" = "application/json"
}

$body = @{
    model = "claude"
    messages = @(
        @{ role = "user"; content = "Write a quicksort function in Rust" }
    )
    timeout = 120
    cwd = "F:\AntiGravity\testgame" # (Optional) Workspace path
} | ConvertTo-Json -Depth 8 -Compress

$response = Invoke-RestMethod -Uri "http://localhost:8080/v1/chat/completions" -Method Post -Headers $headers -Body $body
$response.choices[0].message.content
```

#### Multi-Modal Image/Attachment Request
```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer my-secret-lan-token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini",
    "messages": [
      {
        "role": "user",
        "content": [
          {
            "type": "text",
            "text": "What is in this image?"
          },
          {
            "type": "image_url",
            "image_url": {
              "url": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAUAAAAFCAYAAACNbyblAAAAHElEQVQI12P4//8/w38GIAXDIBKE0DHxgljNBAAO9TXL0Y4OHwAAAABJRU5ErkJggg=="
            }
          }
        ]
      }
    ]
  }'
```

### 2. List Providers (`GET /api/providers`)
Checks the availability of CLI tools on the host machine and returns their status.

#### Request
```bash
curl -H "Authorization: Bearer my-secret-lan-token" http://localhost:8080/api/providers
```

#### Response
```json
[
  {
    "name": "claude",
    "available": true,
    "supports_vision": true
  },
  {
    "name": "gemini",
    "available": true,
    "supports_vision": true
  },
  {
    "name": "openai",
    "available": true,
    "supports_vision": true
  }
]
```

### 3. File Upload (`POST /api/upload`)
Allows uploading file attachments via multipart form data. Returns the server path to reference in prompts.

#### Request
```bash
curl -X POST http://localhost:8080/api/upload \
  -H "Authorization: Bearer my-secret-lan-token" \
  -F "file=@/path/to/localfile.txt"
```

#### Response
```json
{
  "path": "F:\\AntiGravity\\CLIController\\temp_uploads\\uuid_localfile.txt",
  "filename": "uuid_localfile.txt"
}
```

## Python Client Example

You can drop the following utility function into any Python application to call your CLIController server:

```python
import base64
import requests

def call_cli_gateway(
    model: str,
    prompt: str,
    system_prompt: str = None,
    image_path: str = None,
    cwd: str = None,
    token: str = "my-secret-lan-token",
    timeout_secs: int = 120
) -> str:
    url = "http://localhost:8080/v1/chat/completions"
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json"
    }
    
    # 1. Structure the message payload
    user_content = [{"type": "text", "text": prompt}]
    
    if image_path:
        with open(image_path, "rb") as image_file:
            encoded_string = base64.b64encode(image_file.read()).decode("utf-8")
        
        ext = image_path.split(".")[-1].lower()
        mime_type = f"image/{ext}" if ext in ["png", "jpg", "jpeg", "webp", "gif"] else "image/jpeg"
        
        user_content.append({
            "type": "image_url",
            "image_url": {
                "url": f"data:{mime_type};base64,{encoded_string}"
            }
        })
        
    messages = []
    if system_prompt:
        messages.append({"role": "system", "content": system_prompt})
        
    messages.append({"role": "user", "content": user_content})
    
    # 2. Execute Request
    payload = {
        "model": model,
        "messages": messages,
        "timeout": timeout_secs
    }
    if cwd:
        payload["cwd"] = cwd
    
    response = requests.post(url, json=payload, headers=headers, timeout=timeout_secs + 10)
    response.raise_for_status()
    
    return response.json()["choices"][0]["message"]["content"]
```

## Claude Code Skill: "Consult the CLIs"

This repo ships a [Claude Code](https://docs.claude.com/en/docs/claude-code) **Skill** that turns the gateway into a one-phrase command. Once installed, you can just tell your Claude Code agent *"consult the other CLIs"* (or in Vietnamese, *"tham vấn mấy ông CLI"*) and it will automatically forward your question to the local `claude`, `gemini`, and `openai` (Codex) CLIs through the gateway and synthesize the answers.

It lives in [`skills/consult-local-clis/`](skills/consult-local-clis/) and is built around a dependency-free Python client ([`scripts/consult.py`](skills/consult-local-clis/scripts/consult.py)) that handles two common pitfalls automatically: Windows shell quote-mangling on inline `curl`, and the Codex CLI rejecting a system role.

### Install

**Option A — drop-in folder** (works on any OS):
```bash
# Copy the skill into your personal Claude Code skills directory
cp -r skills/consult-local-clis ~/.claude/skills/
```
On Windows that target is `%USERPROFILE%\.claude\skills\`.

**Option B — packaged bundle:** install [`skills/consult-local-clis.skill`](skills/consult-local-clis.skill) via your Claude Code skill installer.

### Use

With the gateway running on `http://localhost:8080`, start a Claude Code session and say things like:
- "tham vấn mấy ông CLI xem nên chọn Postgres hay SQLite cho dự án này"
- "get a second opinion from Codex on this function"
- "ask gemini and claude to review this architecture"

You can also call the client directly:
```bash
python skills/consult-local-clis/scripts/consult.py --all --prompt "What is a mutex?"
python skills/consult-local-clis/scripts/consult.py --model openai --prompt "Refactor this" --timeout 240
python skills/consult-local-clis/scripts/consult.py --providers   # check which CLIs are up
```
Override the gateway location/token with `--base-url` / `--token` or the env vars `CLI_CONTROLLER_URL` / `CLI_CONTROLLER_TOKEN`.

## Multi-Agent Orchestration

Use one desktop AI agent as a **conductor** that plans a task and delegates step-by-step subtasks to the *other* local CLIs through the gateway — keeping a shared understanding across each step. It is **orchestrator-agnostic** (drive it with Gemini/Antigravity, Claude Code, or Codex) and **portable across machines**. (Full write-up: **[ANNOUNCEMENT.md](ANNOUNCEMENT.md)**.)

**How shared context works.** The gateway is stateless and each worker is a one-shot process, so shared memory lives on disk: a ledger file `AGENT_LOG.md` in your working directory that every delegated worker reads first and appends to. Pass your project folder as the request's `cwd` so workers run inside it with real file access.

### Fastest start — one link, zero setup

Open the agent you already use and tell it:

> *"Read `http://localhost:8080/agents` and apply it to this project to orchestrate the other AI CLIs."*

- **`/agents`** returns the full machine-readable protocol — the agent reads it and is ready to delegate.
- **`/consult.py`** lets the agent download the helper with one `curl` — no files need to pre-exist in your project.

### The delegation helper (`consult.py`)

```bash
# Delegate a real task inside a shared workspace (auto-applies the AGENT_LOG.md ledger)
python consult.py --model claude --workspace . --prompt "Implement the parser module"

# Ask a chosen subset (the two strongest, quota-efficient workers)
python consult.py --models claude,openai --prompt "Review this approach"

# gemini/agy is the slowest backend, so give it room
python consult.py --model gemini --timeout 300 --prompt "..."
```

Useful flags: `--workspace <dir>` (run inside a shared dir + apply the ledger protocol), `--models a,b` (subset), `--all`, `--timeout`, `--retries` (auto-retry transient failures; default 2). Download it from `/consult.py` or use [`skills/consult-local-clis/scripts/consult.py`](skills/consult-local-clis/scripts/consult.py).

### Suggested roles

- **claude (Claude Code)** — senior technical advisor; hard design, architecture, and review calls.
- **openai (Codex)** — broad context and overview; implementation, plan coherence, cross-checks.
- **gemini (Antigravity / agy)** — fully capable, but the slowest backend; give it a generous `--timeout` (300+) for work you can wait on.

### Alternative — drop the files in your project

Copy the protocol files from [`orchestration/`](orchestration/) into your project root: `ORCHESTRATION.md` plus the pointer for your conductor — `CLAUDE.md`, `AGENTS.md` (Codex), or `GEMINI.md` (Antigravity). The full protocol (ledger format, context envelope, role guidance, portability rules) lives in [`orchestration/ORCHESTRATION.md`](orchestration/ORCHESTRATION.md).
