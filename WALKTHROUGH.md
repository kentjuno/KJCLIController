# Testing & Completion Guide: CLIController

The **CLIController** project has been written and compiled successfully in Rust at `f:\AntiGravity\CLIController`. Below is a summary of the files created, configuration instructions, and the steps to run the tests.

---

## 1. Components Developed

* **Configuration & Security**:
  * [src/config.rs](src/config.rs): Automatically loads parameters from the `config.json` file (or environment variables) and creates the temporary upload directory and the output directory on startup.
  * [src/auth.rs](src/auth.rs): Bearer Token authentication middleware for LAN testing, using a constant-time comparison to guard against timing attacks.
* **CLI Adapters**:
  * [src/cli/resolve.rs](src/cli/resolve.rs): Resolves the absolute paths of the executables (including npm `.cmd` shim files on Windows) and sets the `CREATE_NO_WINDOW` flag to prevent a black console window from flashing.
  * [src/cli/validate.rs](src/cli/validate.rs): Enforces prompt length (max 100KB) and attachment count (max 10), verifies file read permissions, and blocks path traversal (`..`) attempts.
  * [src/cli/claude.rs](src/cli/claude.rs): Adapter for Claude Code — pipes the prompt over stdin, auto-grants directory access (`--add-dir`), and enables permission bypass.
  * [src/cli/gemini.rs](src/cli/gemini.rs): Adapter for the Agy CLI — runs serially behind a `Semaphore(1)`, exchanges input/output via temp files, strips ANSI, and parses the JSON fragment.
  * [src/cli/codex.rs](src/cli/codex.rs): Adapter for OpenAI Codex — auto-detects the CLI's image attachment flag and supports a dual-mode fallback to the REST API (the `gpt-4o` model with images sent as base64) when the CLI is unavailable or blocked.
* **API Routing**:
  * [src/registry.rs](src/registry.rs): Manages the lifecycle and caches the availability status of the CLIs, performing a fail-fast check before executing a command.
  * [src/main.rs](src/main.rs): Launches the Axum server, parses the OpenAI payload, extracts base64 images to temp storage, scans the `outputs/` directory to collect files/images generated by Codex, and attaches them to the response as Markdown.
* **Chat UI**:
  * [static/index.html](static/index.html): A single-page HTML5 layout with configuration columns and the chat stream.
  * [static/style.css](static/style.css): Dark-mode design (HSL Slate), glassmorphism borders, and smooth chat-bubble entrance effects.
  * [static/app.js](static/app.js): Client-side controller — reads image files into base64 URLs, packages the OpenAI request, and renders the markdown/image responses.

## 2. Run & Test Instructions

### Step 1: Launch the Server
Open a terminal in the project directory and run:
```cmd
cd f:\AntiGravity\CLIController
cargo run
```
*On the first run, the server automatically creates a `config.json` file in the root directory. You can open this file to change the port or the security token string.*

### Step 2: Use the Web Chat UI
1. Open **`http://localhost:8080/`** in your browser.
2. Enter your LAN security token in the **Bearer Authentication Token** field (default: `my-secret-lan-token`).
3. Select the CLI model you want to try (e.g. Claude Code).
4. Type a message, optionally attach image/document files, and click **Send**.

### Step 3: Test OpenAI API Compatibility with cURL
Open another terminal and send the following request (change the token if you edited it in `config.json`):
```bash
# Send an OpenAI-compatible request
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer my-secret-lan-token" \
  -H "Content-Type: application/json" \
  -d "{\"model\":\"claude\",\"messages\":[{\"role\":\"user\",\"content\":\"Say pong\"}]}"
```
*The response will follow the standard OpenAI JSON structure.*

## 3. UI Bug Fix (June 2026)

* **Problem**: The Chat UI rendered but did not work (clicking send reloaded the page, and the host CLI list hung at "Checking host environment...").
* **Cause**: The HTML file [static/index.html](static/index.html) was missing the `<script>` tag that loads the logic file [static/app.js](static/app.js).
* **Fix**: Added `<script src="/static/app.js"></script>` right before the closing `</body>` tag.
* **Result**: The UI works perfectly — it queries the CLI statuses on page load, sends messages asynchronously (AJAX) without reloading the page, and displays responses as chat bubbles.

## 4. Codex CLI Bug Fix (June 2026)

* **Problem**: The Codex CLI did not respond to messages in the Web UI and returned the error `Incorrect API key provided: my-secre*******oken` from the REST API fallback.
* **Cause**:
  1. A previous version overrode the `OPENAI_API_KEY` environment variable with the LAN security token (`my-secret-lan-token`), causing the CLI and API fallback to use the wrong credential.
  2. The Rust server did not yet support configuring a real OpenAI API key via `config.json` to pass to the child process.
* **Fix**:
  1. Completely removed the line that overrode the `OPENAI_API_KEY` environment variable with the LAN token.
  2. Added an `openai_api_key` field to the `AppConfig` configuration struct ([src/config.rs](src/config.rs)) and automatically set the real `OPENAI_API_KEY` environment variable when it is configured in `config.json`.
* **Result**: The Codex CLI works perfectly, responding directly to user questions without 401 or permission-block errors.
